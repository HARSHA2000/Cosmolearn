## What This Lab Is About

Most final year projects end at model training. The model sits in a Jupyter notebook, evaluated on a test set, and never used again. That gap — between a trained model and a deployed product — is exactly what this lab bridges.

In one day, students take a pre-trained ML model (or bring their own final year project model) and deploy it to a live URL with proper containerization, automated deployment, and monitoring. This is what separates a "project" from a "product" on a resume.

## Curriculum

### Session 1 — FastAPI & Model Serving (2 hours)

**The gap between notebook and production**
- Why Flask/Streamlit is not production ML serving
- REST API design for ML: inputs, outputs, versioning
- FastAPI basics: routes, request models, response schemas
- Async endpoints and why they matter at scale

**Hands-on: Wrap your model in a FastAPI app**
- Build a `/predict` endpoint that accepts JSON and returns predictions
- Add `/health` and `/metrics` endpoints
- Local testing with `curl` and the automatic Swagger UI
- Handling errors: invalid inputs, model failures, timeouts

### Session 2 — Containerization with Docker (2 hours)

**Why Docker for ML deployments**
- Dependency isolation — "it works on my machine" → solved
- Image layers, caching, and build optimization
- Docker for ML: dealing with large model files
- Docker Compose for multi-container apps (API + database)

**Hands-on: Dockerize your FastAPI ML app**
- Write an efficient `Dockerfile` for a Python ML project
- Build and run the container locally
- Reduce image size: multi-stage builds, `.dockerignore`
- Push to Docker Hub / GitHub Container Registry

### Session 3 — AWS EC2 Deployment (2 hours)

**Cloud fundamentals for engineers**
- EC2 instance types: what to pick for ML serving
- Security groups, IAM roles, and key-pair SSH
- Nginx as a reverse proxy — routing traffic to your container

**Hands-on: Deploy to a live EC2 instance**
- SSH into an EC2 instance (pre-provisioned)
- Pull Docker image and run the container
- Configure Nginx to proxy `/predict` to the container
- Test the live API endpoint from a browser

### Session 4 — CI/CD and Monitoring (2 hours)

**GitHub Actions for automated deployments**
- What CI/CD means and why it matters
- Writing a GitHub Actions workflow: test → build → push → deploy
- Secrets management: API keys and SSH credentials in GitHub
- Hands-on: Push a code change and watch it auto-deploy

**Monitoring with Prometheus + Grafana**
- What to monitor in an ML deployment: latency, error rate, throughput
- Instrumenting a FastAPI app with `prometheus-fastapi-instrumentator`
- Grafana dashboard basics: panels, queries, alerts
- Hands-on: Set up a live dashboard showing your API's request metrics
